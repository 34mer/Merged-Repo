It owns: • all schemas, • DSL definitions, • canonicalization logic, • hashing, • stable IDs, • lineage/provenance helpers. Nothing else invents its own record format. 2. kb This is Layer 0. It owns: • immutable domain records, • failed conjectures, • representation maps, • trust metadata, • retrieval, • verification recipes, • snapshots. This is the accumulated intelligence of the system. 3. generators This is Layer 1. It owns: • exact computation, • enumeration, • family-specific object generation, • mutation generation, • backend selection. Heavy libraries like Sage/polymake live here as optional adapters. 4. normalize This is Layer 2A. It owns: • deterministic canonicalization, • feature extraction, • model-assisted normalization only where necessary. It must run before proposing. 5. propose This is Layer 2B. It owns: • local small-model inference, • retrieval-augmented prompting, • grammar-constrained decoding, • proposal repair. It never returns raw prose to the rest of the pipeline. 6. check This is Layer 3. It owns: • symbolic checks, • combinatorial checks, • numeric checks, • SMT checks, • optional formal checks. Truth lives here. 7. search This is Layer 4. It owns: • queueing, • novelty scoring, • frontier management, • budgets, • promotion/demotion, • exploration vs exploitation. 8. stress This is Layer 5. It owns: • held-out testing, • symmetry-preserving mutations, • symmetry-breaking mutations, • adversarial near-miss generation, • robustness scoring. 9. substrate This is Layer 6. It owns: • automatic draft compilation of substrate constraints, • schema generation from strong survivors, • ranking by substrate relevance, • cumulative substrate-spec registry. 10. review This is Layer 7 support.
