## 0. Non-Promotion Notice

This protocol enforces math-first workflow only. It does not prove Formation Medium, define the final mathematical object, validate any physical regime, select a substrate, establish engineering readiness, or replace actual reasoning.

## 1. Core Rule

No major source-review, physical-comparator, route, substrate, or broad repo-expansion work may proceed before a Stage 0 math-first attack pass.

Tools serve the math plan.

Tools do not choose the plan.

The reason is explicit:

```text
full project awareness degrades during long tool use
```

So the workflow must force tool-light mathematical orientation before tool-heavy execution.

## 2. Stage Sequence

### Stage 0 — Deep Math Attack

Purpose:

```text
define the actual mathematical object before repo motion
```

Allowed:

```text
read core context
read relevant repo files
local scratch reasoning
small sanity grep
blackboard note if useful
```

Disallowed by default:

```text
broad internet/source shopping
candidate regime expansion
validator-first work
physics analogy matching
large symbolic search
promotion/status mutation
```

Required questions:

```text
What is the actual mathematical object in this step?
Which definitions must be fixed before any computation?
What is the smallest literal finite object to compute?
What theorem, counterexample, or invariant would change the project?
What would show this direction is wrong or irrelevant?
Which existing theorem/source/check is a thin stick?
What should be deleted, demoted, or quarantined if this fails?
```

### Stage 1 — Formal Target Extraction

Allowed:

```text
definitions
typed predicates
finite object schemas
theorem / counterexample statements
formal check target definitions
```

Exit condition:

```text
Exact statements exist for what must be computed or proved, including failure conditions and non-promotion boundaries.
```

### Stage 2 — Literal Computation

Allowed:

```text
Python exact finite checks
Z3 / SMT fixtures
Sage or polymake when available
Lean / Lake theorem skeletons
Ax-Prover / Aristotle proof-search candidates validated only by Lean
```

Exit condition:

```text
A concrete check result, counterexample, theorem skeleton, or failed attempt with artifacted assumptions.
```

### Stage 3 — Emergence Triage

Allowed:

```text
physical-regime scoring
engineering route triage
source review
negative controls
```

Exit condition:

```text
Candidate physics / engineering is derived from or tested against Stage 0–2 math, not introduced by resemblance or convenience.
```
