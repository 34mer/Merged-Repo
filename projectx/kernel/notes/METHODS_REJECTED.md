# Methods Rejected

## Rejected by default
- destructive scan-and-rebuild as assumed transfer
- static snapshot equivalence as assumed continuity
- copied memory as success criterion
- copied behavior as success criterion
- branch creation treated as preserved singular continuity
- product wedge success treated as object completion

## Rejection note
Rejected does not always mean useless.
A rejected method may still yield instruments, data, or boundary information.
It does not satisfy the object by default.
