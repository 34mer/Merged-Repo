# Merged Architecture

`projectx/` is the outer operating kernel. It owns continuity, state, review, evaluation, and control.

`pg_pipeline/` is the inner research subsystem. It owns typed scientific records, deterministic generation, normalization, proposal, checking, stress, substrate compilation, and weekly summaries.

`exact_work/` is the exact-work authority pack. It owns handoff notes, Lean status, Lean files, exact scripts when valid sources are available, and exact outputs when valid sources are available.

`archive/` is not canonical. It stores imported raw source material, old kernel snapshots, corrupt source artifacts, and deprecated copied ProjectX control modules.

Boundary rules:

- Runtime state is not a scientific truth record.
- Artifact files are not typed records until ingested through `pgcore` schemas.
- Review decisions are separate from proposals and substrate drafts.
- Lean is authority-bearing but secondary until the Python lifecycle is alive.
